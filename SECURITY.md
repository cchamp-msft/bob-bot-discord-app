# Security Policy

## Supported versions

Only the latest version of this project is supported with security updates.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Preferred:
- Use GitHub "Report a vulnerability" (Security Advisories / Private vulnerability reporting), if enabled for the repository.

If you cannot use a private advisory:
- Open an issue with minimal details and request a private channel. Avoid including secrets, access tokens, or exploit steps.

## What to expect

- We will acknowledge receipt when possible.
- We will prioritize based on severity and impact.
